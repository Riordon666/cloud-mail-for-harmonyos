# 05 — 写邮件页面完善

> **前置条件**：02 完成
> **对标原项目**：`layout/write/index.vue` + `components/tiny-editor/`

## 任务清单

### 任务 1：收件人输入增强
- [ ] 多个收件人（逗号或分号分隔，自动解析为标签）
- [ ] 收件人标签展示（带 × 删除按钮）
- [ ] 抄送(Cc) + 密送(Bcc) 展开/收起
- [ ] 邮箱格式校验提示

### 任务 2：主题与正文
- [ ] 主题输入框（带字数限制提示）
- [ ] 正文编辑区：
  - 方案 A：`TextArea`（纯文本，简单）
  - 方案 B：`RichEditor`（富文本，对标 TinyMCE，但不支持 HTML 源码编辑）
  - 建议先 TextArea，后续再升级

### 任务 3：附件处理
- [ ] "添加附件"按钮（SymbolGlyph paperclip）
- [ ] 文件选择器（DocumentViewPicker）→ 选择文件
- [ ] 附件芯片展示（文件名 + 大小 + × 删除）
- [ ] 附件数量限制（最多 10 个）
- [ ] 文件大小校验（单文件 < 25MB）

### 任务 4：发送功能（Mock）
- [ ] 发送按钮（顶部右侧，SymbolGlyph paperplane）
- [ ] 发送前校验：收件人不为空、主题不为空
- [ ] 发送中状态：按钮 loading + Toast 提示
- [ ] 发送成功：Toast "发送成功" + 清空表单 + 跳回收件箱
- [ ] 发送失败：Toast 错误信息
- [ ] 此阶段使用 mock 延迟模拟，不调真实 API

### 任务 5：草稿自动保存
- [ ] 每 30 秒自动保存草稿到 Preferences
- [ ] 页面退出时保存草稿
- [ ] 重新进入时恢复草稿（提示是否继续编辑）
- [ ] 发送成功后删除草稿

### 任务 6：回复/转发模式
- [ ] 接受路由参数判断模式（新建/回复/转发）
- [ ] 回复模式：自动填充收件人、主题（Re: xxx）、引用原文
- [ ] 转发模式：自动填充主题（Fwd: xxx）、引用原文

## 验收标准
- ✅ 多人收件人输入
- ✅ 附件选择与展示
- ✅ 发送 mock 流程完整
- ✅ 草稿自动保存与恢复
- ✅ 回复/转发模式正常
