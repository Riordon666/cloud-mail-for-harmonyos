# 04 — 状态管理

> ArkUI 状态管理对标 Vue 3 响应式系统 + Pinia

## 状态装饰器对照表

| ArkUI | Vue 3 对标 | 用途 | 数据流 |
|-------|-----------|------|--------|
| `@State` | `ref()` / `reactive()` | 组件内部状态 | 组件内 |
| `@Prop` | `props` | 父→子单向 | 单向 |
| `@Link` | `v-model` | 父↔子双向 | 双向 |
| `@Provide` | `provide()` | 祖先提供 | 向下 |
| `@Consume` | `inject()` | 后代消费 | 向下 |
| `@StorageLink` | Pinia store | 全局双向 | 全局 |
| `@StorageProp` | Pinia (readonly) | 全局单向 | 全局 |
| `@Observed` | — | 类属性观察 | — |
| `@ObjectLink` | `storeToRefs()` | 对象属性双向 | 双向 |
| `@Watch` | `watch()` | 变化回调 | — |

## @State — 组件本地状态

```typescript
@Component
struct InboxView {
  @State emails: EmailData[] = []
  @State searchKeyword: string = ''
  @State isLoading: boolean = false
  @State currentTab: number = 0

  build() {
    List() {
      ForEach(this.emails, (item: EmailData) => {
        // 修改 emails 时自动刷新 UI
      })
    }
  }

  async loadEmails() {
    this.emails = await EmailApi.fetchList()  // 赋值即触发 UI 更新
  }
}
```

**注意事项**：
- 数组修改后需 **整体替换** 才能触发更新：`this.emails = [...this.emails, newItem]`
- 对象修改需整体替换或使用 `@Observed` + `@ObjectLink`
- `@State` 仅作用当前组件，不传递给子组件

## @Prop vs @Link — 父子通信

```typescript
// 父组件
@Component
struct Parent {
  @State count: number = 0

  build() {
    Column() {
      ChildProp({ value: this.count })        // 单向: 父→子
      ChildLink({ value: $count })             // 双向: 父↔子（$语法）
    }
  }
}

// 子组件 — 单向接收
@Component
struct ChildProp {
  @Prop value: number    // 父变→子变；子变不影响父
  build() { Text(`${this.value}`) }
}

// 子组件 — 双向绑定
@Component
struct ChildLink {
  @Link value: number    // 父变↔子变
  build() {
    Button('+1').onClick(() => { this.value++ })
  }
}
```

**使用场景**：
- `@Prop`: 展示型子组件（如 EmailCard 只需要显示数据）
- `@Link`: 交互型子组件（如写邮件的 TextInput 需要回写父组件状态）

## @Provide / @Consume — 跨层级传递

```typescript
// 祖先组件（不需要中间组件逐层传递）
@Component
struct AppRoot {
  @Provide('userInfo') userInfo: UserData = { name: '', email: '' }

  build() {
    HdsNavigation() {
      DeepChild()   // 无需显式传 props 给 DeepChild
    }
  }
}

// 深层后代
@Component
struct DeepChild {
  @Consume('userInfo') userInfo: UserData   // 自动接收
}
```

## @StorageLink / @StorageProp — 全局状态

```typescript
// 写入全局状态
AppStorage.setOrCreate('colorMode', 0)
AppStorage.setOrCreate('token', 'xxx-yyy-zzz')

// 组件中读取
@Component
struct MyComponent {
  @StorageLink('colorMode') colorMode: number = 0     // 读写双向
  @StorageProp('token') token: string = ''             // 只读

  private isDark(): boolean {
    return this.colorMode === 1
  }
}
```

**Cloud Mail 使用模式**：
```typescript
// ThemeStore.ets — 全局主题管理
export class ThemeStore {
  static setDarkMode(isDark: boolean) {
    AppStorage.setOrCreate('colorMode', isDark ? 1 : 0)
  }

  static toggleMode() {
    const current = AppStorage.get<number>('colorMode') ?? 0
    AppStorage.setOrCreate('colorMode', current === 0 ? 1 : 0)
  }
}

// UserStore.ets — 全局用户信息
export class UserStore {
  static setUser(user: UserData) {
    AppStorage.setOrCreate('userInfo', user)
    AppStorage.setOrCreate('token', user.token)
  }

  static logout() {
    AppStorage.setOrCreate('token', '')
    AppStorage.setOrCreate('userInfo', {})
  }
}
```

## @Observed / @ObjectLink — 类属性观察

```typescript
@Observed
class EmailData {
  emailId: number
  subject: string
  unread: boolean
  attList: Attachment[]

  constructor(id: number, subject: string) {
    this.emailId = id
    this.subject = subject
    this.unread = true
    this.attList = []
  }

  markRead() { this.unread = false }
}

@Component
struct EmailCard {
  @ObjectLink email: EmailData   // 对象属性变化自动触发刷新

  build() {
    Row() {
      Text(this.email.subject)
      if (this.email.unread) { Circle().width(8).height(8) }
    }
    .onClick(() => { this.email.markRead() })  // 点击后蓝点消失
  }
}
```

## @Watch — 状态变化回调

```typescript
@Component
struct SearchPage {
  @State @Watch('onSearchChange') keyword: string = ''

  onSearchChange() {
    if (this.keyword.length > 0) {
      this.debounceSearch()         // 输入变化 → 自动搜索
    }
  }

  private timer: number = 0
  private debounceSearch() {
    clearTimeout(this.timer)
    this.timer = setTimeout(() => { this.doSearch() }, 300)
  }
}
```
