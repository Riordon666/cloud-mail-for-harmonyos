# 06 — 数据持久化

> ArkData 对标原项目 D1 + KV + Drizzle ORM

## 三种持久化方案

| 方案 | 对标原项目 | 数据类型 | 容量 | 适用场景 |
|------|-----------|---------|------|---------|
| **Preferences** | 无直接对标 | 键值对 | 小 | 用户设置、Token |
| **KV-Store** | Cloudflare KV | 键值对 | 中 | 缓存、临时数据 |
| **RelationalStore** | D1 + Drizzle | SQLite | 大 | 邮件、账号、联系人 |

---

## Preferences — 轻量键值存储

最适用户设置、Token、草稿等小数据。

```typescript
import { preferences } from '@kit.ArkData'

// 获取实例
const pref = preferences.getPreferencesSync(this.context, { name: 'mail_settings' })

// 写入
pref.putSync('token', 'xxx-yyy-zzz')
pref.putSync('colorMode', 1)
pref.putSync('fontSize', 14)
pref.flushSync()   // 同步刷盘

// 读取
const token: string = pref.getSync('token', '') as string
const colorMode: number = pref.getSync('colorMode', 0) as number

// 删除
pref.deleteSync('draft_email_123')
pref.flushSync()

// 监听变化
pref.on('change', (key: string) => {
  console.log(`${key} changed`)
})
```

**Cloud Mail 使用场景**：
```typescript
// 对标原项目 pinia-plugin-persistedstate
export class LocalStore {
  static saveDraft(emailId: number, draft: DraftData) {
    const pref = preferences.getPreferencesSync(getContext(), { name: 'drafts' })
    pref.putSync(`draft_${emailId}`, JSON.stringify(draft))
    pref.flushSync()
  }

  static getToken(): string {
    const pref = preferences.getPreferencesSync(getContext(), { name: 'auth' })
    return pref.getSync('token', '') as string
  }

  static saveSettings(settings: UserSettings) {
    const pref = preferences.getPreferencesSync(getContext(), { name: 'settings' })
    pref.putSync('autoRefresh', settings.autoRefresh)
    pref.putSync('notifyEnabled', settings.notifyEnabled)
    pref.flushSync()
  }
}
```

## RelationalStore — SQLite 关系数据库

对标原项目 D1 + Drizzle ORM。用于邮件数据、账号列表等结构化数据。

```typescript
import { relationalStore } from '@kit.ArkData'

// 创建/打开数据库
const STORE_CONFIG: relationalStore.StoreConfig = {
  name: 'CloudMail.db',
  securityLevel: relationalStore.SecurityLevel.S2  // 加密级别
}

const store = await relationalStore.getRdbStore(this.context, STORE_CONFIG)

// 建表
await store.executeSql(`
  CREATE TABLE IF NOT EXISTS email (
    email_id INTEGER PRIMARY KEY AUTOINCREMENT,
    send_email TEXT,
    subject TEXT,
    content TEXT,
    type INTEGER DEFAULT 0,
    status INTEGER DEFAULT 0,
    unread INTEGER DEFAULT 0,
    account_id INTEGER,
    user_id INTEGER,
    create_time TEXT DEFAULT CURRENT_TIMESTAMP,
    is_del INTEGER DEFAULT 0
  )
`)

// 插入（使用参数化防注入）
const valueBucket: relationalStore.ValuesBucket = {
  send_email: 'alice@example.com',
  subject: 'Q4 设计评审',
  content: '<p>邮件正文</p>',
  type: 0,      // 0=收件 1=发件
  unread: 0
}
await store.insert('email', valueBucket)

// 查询（带条件、分页）
const predicates = new relationalStore.RdbPredicates('email')
predicates.equalTo('user_id', userId)
  .and().equalTo('type', 0)
  .and().equalTo('is_del', 0)
  .orderByDesc('email_id')
  .limit(20)

const resultSet = await store.query(predicates)
while (resultSet.goToNextRow()) {
  const emailId = resultSet.getLong(resultSet.getColumnIndex('email_id'))
  const subject = resultSet.getString(resultSet.getColumnIndex('subject'))
  // ...
}
resultSet.close()

// 更新
const updateBucket: relationalStore.ValuesBucket = { unread: 1 }
const updatePred = new relationalStore.RdbPredicates('email')
updatePred.equalTo('email_id', emailId)
await store.update(updateBucket, updatePred)

// 删除（软删除）
const deleteBucket: relationalStore.ValuesBucket = { is_del: 1 }
const deletePred = new relationalStore.RdbPredicates('email')
deletePred.in('email_id', [1, 2, 3])
await store.update(deleteBucket, deletePred)

// 事务
store.beginTransaction()
try {
  await store.insert('email', emailData)
  await store.insert('attachments', attData)
  store.commit()
} catch (err) {
  store.rollBack()
}
```

**数据库加密**：
```typescript
// S1-S4 加密级别，S2 为推荐

// S1: 低安全  S2: 中等  S3: 高  S4: 最高
```

## KV-Store — 分布式键值存储

对标原项目 Cloudflare KV。支持跨设备同步。

```typescript
import { distributedKVStore } from '@kit.ArkData'

// 创建 KV 管理器
const kvManager = distributedKVStore.createKVManager({ bundleName: 'com.example.cloudmail' })
const options = {
  createIfMissing: true,
  encrypt: true,
  backup: true,
  kvStoreType: distributedKVStore.KVStoreType.SINGLE_VERSION
}
const kvStore = await kvManager.getKVStore('mail_cache', options)

// 写入
await kvStore.put('last_sync_time', Date.now().toString())

// 读取
const lastSync = await kvStore.get('last_sync_time')

// 删除
await kvStore.delete('temp_upload_key')
```

## 数据存储决策矩阵

| 数据类型 | 存储方案 | 理由 |
|---------|---------|------|
| 用户 Token | Preferences | 小数据、频繁读取 |
| 用户设置（外观/通知） | Preferences | 小数据、独立键 |
| 邮件列表数据 | RelationalStore | 结构化、大量数据、需要查询 |
| 邮件详情缓存 | RelationalStore | 关联查询（邮件+附件） |
| 附件索引 | RelationalStore (att 表) | 关联邮件 ID |
| 附件文件 | Core File Kit (filesDir) | 二进制大文件 |
| 草稿 | Preferences (小) 或 RelationalStore | 取决于大小 |
| 最近搜索 | Preferences | 少量字符串 |
| 离线缓存 | KV-Store | 支持过期 TTL |

## 对标原项目 schema 迁移

原项目 Drizzle schema → RelationalStore SQL：

```sql
-- user 表
CREATE TABLE user (
  user_id INTEGER PRIMARY KEY AUTOINCREMENT,
  email TEXT NOT NULL,
  type INTEGER DEFAULT 1,
  password TEXT NOT NULL,
  salt TEXT NOT NULL,
  status INTEGER DEFAULT 0,
  send_count INTEGER DEFAULT 0,
  create_time TEXT DEFAULT CURRENT_TIMESTAMP,
  is_del INTEGER DEFAULT 0
);

-- account 表
CREATE TABLE account (
  account_id INTEGER PRIMARY KEY AUTOINCREMENT,
  email TEXT NOT NULL,
  name TEXT DEFAULT '',
  user_id INTEGER NOT NULL,
  all_receive INTEGER DEFAULT 0,
  sort INTEGER DEFAULT 0,
  is_del INTEGER DEFAULT 0
);

-- attachments 表
CREATE TABLE attachments (
  att_id INTEGER PRIMARY KEY AUTOINCREMENT,
  email_id INTEGER NOT NULL,
  filename TEXT,
  mime_type TEXT,
  size INTEGER,
  key TEXT NOT NULL,
  type INTEGER DEFAULT 0,  -- 0=普通附件 1=内嵌图片
  create_time TEXT DEFAULT CURRENT_TIMESTAMP
);
```
